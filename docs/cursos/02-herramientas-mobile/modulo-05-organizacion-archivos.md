## 1. Por qué la Organización es Crucial

### El Problema del CM

```
REALIDAD DEL COMMUNITY MANAGER:

- 50+ productos de joyería = 200+ fotos/videos
- 4-6 formatos de contenido = más archivos
- Múltiples semanas de contenido = más archivos
- Varios canales (IG, FB, Pinterest) = más archivos
- Clientes pidiendo "esa foto que mostraba..." = más archivos

SI NO ESTÁS ORGANIZADA:

❌ "Dónde está esa foto del anillo que publiqué hace 3 meses?"
❌ "Ese video que editamos, dónde quedó?"
❌ "Tenía una versión mejor del post, ahora cuál tengo?"
❌ "Cuántas versiones de este diseño tengo?"
❌ "Ese caption que escribí, dónde lo guardé?"

RESULTADO: Horas perdidas buscando archivos.
           Frustración. Contenido repetido o perdido.
```

### El Beneficio de la Organización

```
BENEFICIOS MEDIBLES:

TIEMPO:
- Buscar archivo: 30 segundos vs 10 minutos
- Preparar sesión de contenido: 15 min vs 45 min
- Crear variaciones: 5 min vs 20 min

CALIDAD:
- Siempre usar la versión correcta
- No duplicar trabajo
- Consistencia visual por acceso fácil

PRESIÓN MENTAL:
- Saber qué tienes y dónde está
- Menos ansiedad por "qué voy a publicar"
- Fácil crear contenido nuevo desde existentes

EFICIENCIA:
- Backlog organizado = fácil acceder
- Referencias guardadas = rápido crear
- Templates organizados = rápido diseñar
```

## 3. Nomenclatura de Archivos

### Sistema de Nomenclatura

```
NOMENCLATURA ESTÁNDAR:

FORMATO BASE:
[Tipo]_[Producto]_[Variante]_[Fecha].[ext]

EJEMPLOS:

Para Fotos:
Post_AnilloLuna_01_20260517.jpg
Post_AnilloLuna_02_20260517.jpg
Post_CollarSol_Cierre_20260517.jpg
Raw_AnilloLuna_01_20260517.jpg
Raw_AnilloLuna_02_20260517.jpg

Para Videos:
Reel_AnilloLuna_v1_20260517.mp4
Reel_AnilloLuna_v2_20260517.mp4
Reel_CollectionPrimavera_20260520.mp4

Para Posts Finales:
Final_Post_AnilloLuna_01_20260517.png
Final_Post_Carrusel_Estilos_01_20260517.png

Para Templates:
Template_Post_Producto_v1.png
Template_Carrusel_05Piezas.png
```

### Guía de Nomenclatura

```
REGLAS:

1. SIN ESPACIOS:
   - Usar guiones bajos (_) o guiones (-)
   - Anillo Luna.jpg ❌ → Anillo_Luna.jpg ✅

2. TODO EN MINÚSCULAS (opcional pero recomendado):
   - Evitar problemas de mayúsculas en sistemas
   - anillo_luna_01.jpg ✅

3. FECHA AL FINAL (para chronological order):
   - 20260517 (YYYYMMDD)
   - Si usas nombre antes: filename_20260517.jpg

4. VERSIONES CLARAS:
   - v1, v2, _v3
   - _FINAL, _DRAFT

5. EXTENSIÓN CORRECTA:
   - .jpg para fotos comprimidas
   - .png para gráficos
   - .mp4 para videos
   - .mov para videos Apple

EVITAR:
- Nombres como "IMG_1234.jpg"
- "Foto1.jpg", "Foto2.jpg"
- "Nueva versión última.jpg"
- "Sin título.png"
```

### Template de Nomenclatura

```
PARA RECORDAR:

┌────────────────────────────────────────────────────┐
│                    NOMENCLATURA                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  [TIPO]_[PRODUCTO]_[DETALLE]_[FECHA].[ext]       │
│                                                    │
│  TIPOS:                                            │
│  - Post (post de Instagram)                        │
│  - Reel (video de Instagram)                       │
│  - Story (historia)                                │
│  - Raw (foto/video original sin editar)           │

│                                                    │
│  DETALLE (opcional):                              │
│  - 01, 02, 03 para versiones                      │
│  - _v1, _v2 para variantes                        │
│  - _editado para indicar estado                    │
│                                                    │
│  FECHA (YYYYMMDD):                                │
│  - Siempre al final                               │
│  - Orden automático en archivos                   │
│                                                    │
└────────────────────────────────────────────────────┘
```

## 6. Búsqueda y Recuperación

### Sistema de Búsqueda

```
BÚSQUEDA EFICIENTE EN GOOGLE DRIVE:

1. SEARCH BAR:
   - Escribe nombre de archivo
   - Búsqueda rápida por palabras clave

2. FILTERS:
   - Tipo: imagen, video, documento
   - Ubicación: carpeta específica
   - Modificado: rango de fechas

3. ADVANCED SEARCH:
   - "type:image" → solo imágenes
   - "owner:me" → solo tus archivos
   - Combinar: "type:video joyeria"

PRÁCTICA:
- Escribe "Anillo" → aparecen todos los archivos de anillos
- Escribe "Post" → aparecen todos los posts
- Escribe "20260517" → aparecen archivos de esa fecha
```

### Tags y Etiquetas

```
EN iOS (Files app):

Crear tags:
1. Abre archivo
2. Toca información (i)
3. Tags > Add Tag
4. Crea tags personalizados:
   - Producto
   - Final
   - Para_Repost
   - Oferta

FILTRAR POR TAG:
- En Files, toca Tags
- Selecciona tag
- Ver todos archivos con ese tag

EN GOOGLE DRIVE (no tiene tags nativos):
- Usar carpetas o subcarpetas
- Prefijo en nombre: [PRD]Anillo_Luna
- Organizar por color (estrellitas)
```

### Versiones y Recuperación

```
GOOGLE DRIVE:

Historial de versiones:
1. Click derecho en archivo
2. "Manage versions"
3. Ver versiones anteriores
4. Descargar versión anterior si necesitas

CANVA:
- Versiones guardadas automáticamente
- File > Version history

CAPCUT:
- Guardar proyecto (no solo exportar video)
- En CapCut, proyecto = editable

RESPALDO:
- Regularmente, guarda proyectos exportados
- Si puedes, usa versión Pro para más storage
```

## 7. Backup Strategy

### Sistema de Backup

```
REGLA 3-2-1:

3 copias de archivos importantes
2 diferentes medios/formats
1 copy offsite (en la nube)

PARA JOYERÍA CM:

COPIA 1: Teléfono/Galería
- Fotos tomadas
- Videos grabados
- RAW files

COPIA 2: Google Drive
- Upload automático de fotos
- Contenido creado
- Todo organizado

COPIA 3: (Opcional) Computer o disco externo
- Para contenido muy importante
- Proyectos de Canva/CapCut (source files)
- Si trabajas también desde PC

AUTOMATIZAR:

En teléfono:
- Google Photos: backup automático de galería
- Configurar: Photos > Settings > Backup
- Asegurar que está activo

Para contenido creado:
- Subir manualmente después de crear
- Crear hábito: "si lo creé, lo subo"
- Revisión semanal de backup
```

### Checklist de Backup

```
CHECKLIST SEMANAL:

□ Fotos raw subida a Drive ✓
□ Contenido creado exportado ✓
□ Proyectos de Canva guardados (export) ✓
□ Videos de CapCut exportados ✓
□ Verificar Google Photos está sincronizando ✓
□ Revisar espacio disponible ✓

CHECKLIST MENSUAL:

□ Limpiar cache de apps
□ Eliminar duplicados
□ Verificar almacenamiento Drive
□ Archive contenido de >3 meses ago
□ Test restore from backup (random sample)
```


## ❌ Common Mistakes

### Mistake 1: Estructura Demasiado Profunda

**Problema:** 6 niveles de carpetas, imposible navegar
**Resultado:** No usas el sistema, vuelves al caos
**Solución:** Máximo 3-4 niveles. Si necesitas más, simplificar.

### Mistake 2: Mezclar Todo en un Lugar

**Problema:** Todo en una carpeta "CM Content"
**Resultado:** 500 archivos sin orden, imposible encontrar nada
**Solución:** Estructura clara, separación por tipo/fecha.

### Mistake 3: Nombres Inconsistentes

**Problema:** Mezclar "_" y "-" y espacios
**Resultado:** No funciona search, duplicados
**Solución:** Elegir formato y stick to it. Ver checklist arriba.

### Mistake 4: No Hacer Backup

**Problema:** Confiar solo en teléfono o 1 lugar
**Resultado:** Si pierdes teléfono, pierdes todo
**Solución:** Google Photos backup ON. Upload a Drive after creation.

### Mistake 5: No Documentar el Sistema

**Problema:** Crear estructura y olvidarla en 2 semanas
**Resultado:** Vuelves a caos porque no recuerdas el sistema
**Solución:** Guardar screenshot de estructura. Agregar a notas.
